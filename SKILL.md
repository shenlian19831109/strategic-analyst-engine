# strategic-analyst-engine Skill (V3.0)

## 1. 概述

`strategic-analyst-engine` 是一个专业级的战略分析引擎，旨在将战略分析从“报告模板生成器”升级为可编排、可推导、可白箱化的智能系统。它通过 **五层知识图谱架构** 与 **横向推理逻辑**，确保每一个战略结论都具备严密的证据链支撑。

## 2. 战略分析框架 V3.0

引擎采用 8 大模块分层递进，形成“分析→洞察→行动”的完整闭环：

| 模块编号 | 模块名称 | 核心回答命题 | 分析方法与工具 |
| :--- | :--- | :--- | :--- |
| **M1** | 行业定义与规模测算 | 盘子多大？风往哪吹？ | TAM/SAM/SOM、渗透率曲线、CAGR |
| **M2** | 宏观环境 PESTEL | 外部红利与红线 | 政策量化映射、经济敏感度、技术成熟度 |
| **M3** | 行业价值链 | 钱在哪？谁说了算？ | 利润池拆解、链主识别、KSF |
| **M4** | 客户需求深度解剖 | 用户到底为什么买单？ | JTBD、需求优先级、痛点/爽点挖掘、沉默需求挖掘 |
| **M5** | 竞争分析与反应模式 | 对手下一步会怎么动？ | 集中度、战略群组、竞争人格编码、反应触发分析 |
| **M6** | 趋势分析 | 技术颠覆与商业模式演进 | 技术成熟度曲线、不确定性矩阵 |
| **M7** | 战略布局图 (ERRC) | 蓝海在哪？怎么创新？ | 价值曲线、消除-减少-提升-创造 |
| **M8** | 商业画布 | 如何落地赚钱？ | 九要素填充、一致性检查、财务可行性 |

### 关键逻辑增强：
*   **M4 前移**：客户需求深度解剖成为竞争分析之前的关键输入，确保“以客户为中心”。
*   **M5 增强**：新增“竞争人格”与“反应模式画像”，从静态份额转向动态博弈。
*   **M6 独立**：趋势分析作为独立模块，为 ERRC 提供前瞻性的技术与模式输入。

## 3. 五层知识图谱架构

### 3.1 逻辑节点层级
*   **L1 战略命题**：核心回答的战略问题（如：行业进入策略）。
*   **L2 分析模块**：框架中的 8 大模块。
*   **L3 分析任务**：模块下的具体分析动作（如：竞争人格编码）。
*   **L4 数据需求**：完成任务所需的量化指标或文本数据。
*   **L5 证据源**：数据的真实出处及可信度（🟢 高、🟡 中、🔴 低）。

### 3.2 跨模块推理规则 (硬编码触发机制)
| 触发器 | 触发条件 | 自动动作 |
| :--- | :--- | :--- |
| **M4 客户痛点** | 识别出高优先级隐形痛点 | 自动更新 M7 ERRC 的“创造”要素建议 |
| **M5 竞争人格** | 识别出“价格杀手”人格 | 自动在 M8 风险评估中增加“价格战预警” |
| **M2 补贴政策** | 政策出现重大修订或新标准 | 重新测算 M3 价值链各环节利润池 |
| **M6 技术成熟度** | 技术逼近“期望膨胀期峰值” | 在 M7 中标记为“高风险需验证” |

## 4. 执行路径：三步闭环

1.  **图谱初始化**：加载 8 大模块、25+ 个标准任务及推理规则。
2.  **数据缺口扫描与采集**：遍历 L3→L4 路径，识别缺失的 L5 证据，自动触发定向采集。
3.  **证据链闭环与报告生成**：每个结论必须挂载 `informs` 关系指向 L5 证据，确保“白箱化”。

## 5. 工程结构

```text
strategic-analyst-engine/
├── SKILL.md                  # 核心方法论手册
├── logic_manifest.json       # 图谱节点与关系的结构化定义
├── templates/                # 标准化输出模板 (JTBD, ERRC, Competitor Radar)
├── knowledge_graph/          # 图谱初始化与缺口分析脚本
└── examples/                 # 行业案例库 (如：中国充电桩行业)
```

## 6. 使用指南

当启动战略分析任务时，Agent 必须：
1.  首先调用 `logic_manifest.json` 明确任务依赖。
2.  针对每个模块，按照“需求点-痛点-爽点”逻辑进行深度挖掘。
3.  在竞争分析中，必须输出竞争对手的“人格画像”和“反应触发模式”。
4.  最终交付物必须包含“数据审计与事实清单”章节。
