# Strategic Analyst Engine (V3.0)

一个专业级的战略分析引擎，融合了 PESTEL、价值链、波特五力、JTBD 需求解剖、竞争反应画像等深度分析方法论，并通过知识图谱实现白箱化推演。

---

## 🏛️ 核心架构：V3.0 战略分析框架

本引擎不仅是一个分析模板，而是一套具备**硬编码推理能力**的智能大脑。它将战略分析拆解为 8 大核心模块，并通过 5 层知识图谱实现逻辑闭环。

### 1. 8 大核心模块
*   **M1: 行业定义与规模测算** - 界定赛道边界，通过 TAM-SAM-SOM 模型确定市场天花板。
*   **M2: 宏观环境 (PESTEL)** - 识别外部环境的“红利”与“红线”，将定性政策转化为定量约束。
*   **M3: 行业价值链分析** - 拆解利润池，识别产业链的权力中心（链主）。
*   **M4: 客户需求深度解剖** - 基于 **JTBD (Jobs-to-be-Done)**、马斯洛需求层次及双因素理论，深挖“需求点-痛点-爽点”。
*   **M5: 竞争分析与反应画像** - 识别 TOP 玩家的“竞争人格”（如价格杀手、技术领跑者）及其反应触发模式。
*   **M6: 趋势分析与技术演进** - 捕捉行业拐点，为战略布局提供前瞻性输入。
*   **M7: 战略布局图 (ERRC)** - 基于需求与竞争画像，寻找“非竞争性”的蓝海空间。
*   **M8: 商业画布 (Business Model Canvas)** - 将所有洞察整合为一套可落地的商业闭环。

---

## 🕸️ 知识图谱驱动 (Knowledge Graph Driven)

本引擎的核心灵魂在于其**五层逻辑链**，确保每一个结论都具备“白箱化”的可追溯性：

1.  **L1 战略命题 (Proposition)**：核心分析目标。
2.  **L2 分析模块 (Module)**：上述 8 大核心模块。
3.  **L3 分析任务 (Task)**：模块下的具体执行动作（如：竞争人格编码）。
4.  **L4 数据需求 (Data Requirement)**：完成任务所需的具体指标。
5.  **L5 证据源 (Evidence)**：数据的真实出处，带置信度评分（🟢/🟡）。

### 🔗 跨模块推理规则
引擎内置了自动触发逻辑，例如：
*   **痛点洞察 $\rightarrow$ 战略创造**：识别出用户的隐形痛点时，自动更新 ERRC 模型的“创造”维度。
*   **竞争人格 $\rightarrow$ 风险预警**：识别出“价格杀手”人格时，自动在商业画布中增加价格战预警。

## 🔍 数据审计与透明度

每一份由本引擎生成的报告都遵循**“证据链闭环”**原则：
*   **数据来源**：所有核心数据均附带原始链接。
*   **推导逻辑**：从数据到结论的每一步推导过程均在图谱中可见。
*   **置信度标注**：对不同来源的数据进行可信度分级。
